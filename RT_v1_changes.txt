------------------------------------------------------------------------
C-01  Surface_Brightness_CLOUDY  [가장 중요한 변경]
------------------------------------------------------------------------
기존 방식:
  - 70개의 projected radius R 각각에 대해 scipy.integrate.quad() 를 호출해
    적분  emis(r)*r/sqrt(r^2-R^2)  을 r=R 부터 r_max 까지 수행.
  - 피적분함수가 r=R 에서 1/sqrt(r-R) 특이점(singularity)을 가지고 있어,
    quad 가 내부적으로 이를 처리하느라 호출당 ~1000회의 함수 평가 발생.
  - 각 quad 호출마다 Python lambda + scipy interp1d 가 실행됨.
  - 전체: 70회 quad × 3종류(CIV, HeII, Lya) = 210회 Python 적분 루프
    (CLOUDY_data_path() 1회 호출 기준).

새로운 방식 (RT_v1.py):
  - 치환  r = sqrt(R^2 + t^2),  dr = t/r dt  를 적용.
    치환 후 피적분함수가  emis(sqrt(R^2+t^2)) dt  로 변환되어
    특이점이 수학적으로 완전히 소멸됨.
    균일 간격 사다리꼴 공식(trapezoid, n_pts=512)으로 정확하게 수렴.
  - 70개 projected radius 루프를 numba @njit(parallel=True) + prange 로
    멀티코어 병렬 실행.
  - np.interp() (numba 네이티브 함수) 로 보간 → Python 오버헤드 없음.
  - cache=True 로 컴파일 결과를 디스크에 저장, 첫 호출 이후는 즉시 실행.

수학적 동치 증명:
  r = sqrt(R^2 + t^2) 으로 치환하면:
    dr = t / sqrt(R^2 + t^2) dt
    r / sqrt(r^2 - R^2) = sqrt(R^2+t^2) / t
  곱하면:
    r/sqrt(r^2-R^2) * dr = sqrt(R^2+t^2)/t * t/sqrt(R^2+t^2) dt = dt
  따라서:
    ∫ emis(r)*r/sqrt(r^2-R^2) dr  =  ∫ emis(sqrt(R^2+t^2)) dt
  로 완전히 동치 (수치 결과 동일).

n_pts 파라미터:
  Surface_Brightness_CLOUDY 함수에 n_pts 인자 추가 (기본값 512).
  emissivity 프로파일이 매우 가파를 경우 1024 또는 2048 로 늘릴 것.

------------------------------------------------------------------------
C-02  photon_number_SB
------------------------------------------------------------------------
기존 방식:
  - 반지름 배열(n ~ 100 원소)에 대해 동일한 for-loop을 두 번 반복.
  - 두 루프 모두 annular 면적을 계산하는 코드가 중복되어 있었음.

새로운 방식:
  - 면적을 numpy 벡터화 연산으로 한 번에 계산.
  - 경계 원소(인덱스 0, -1)만 scalar 대입으로 처리.
  - _photon_number_kernel() 을 @njit 으로 컴파일하여 파라미터 그리드 순환
    시 반복 호출 비용 절감.
  - 중복 면적 계산 완전 제거.

------------------------------------------------------------------------
C-03  _read_ele_column
------------------------------------------------------------------------
기존 방식:
  - Python for-loop 으로 파일을 한 줄씩 읽음.
  - num (= len(Mod.nH)) 을 미리 알아야 배열 사전할당 가능.
  - 거의 동일한 함수가 두 곳에 따로 정의되어 있었음:
      (a) CLOUDY_data_path 내부의 클로저 (인자 3개, num은 외부 변수)
      (b) 파일 하단의 모듈 수준 함수 (인자 4개)
  - 두 버전이 서로 다른 시그니처를 가져 유지보수 위험 존재.

새로운 방식:
  - 단일 모듈 수준 함수 _read_ele_column(path, ele, col_idx) 로 통합.
  - numpy.loadtxt (C 레벨 I/O) 사용 → Python readline 루프 대비 ~10배 빠름.
  - num 인자 불필요 (loadtxt가 모든 행을 자동으로 읽음).
  - CLOUDY_data_path 와 read_file 모두 동일한 함수 사용.

------------------------------------------------------------------------
C-04  SED_properties — 1 Rydberg 인덱스 탐색
------------------------------------------------------------------------
기존 방식:
  idx_1ryd = np.argmin(np.abs(Ryd - 1.0))   # O(N) 전체 스캔

새로운 방식:
  idx_1ryd = np.searchsorted(Ryd, 1.0)       # O(log N) 이진 탐색

주의: Ryd 배열이 오름차순으로 정렬되어 있어야 함 (CLOUDY SED 출력의 표준).
절대적인 시간 개선은 작지만 정렬된 데이터에 대해 올바른 알고리즘 사용.

------------------------------------------------------------------------
C-05  SED_properties — Gaussian Fitting Flux
------------------------------------------------------------------------
기존 방식:
  Python 클로저 Flux(Fc, F_max, lam, lam_c, V_emit) 를 정의하고
  전체 lambda 배열에 한 번 호출. 클로저 호출 오버헤드 존재.

새로운 방식:
  numpy 표현식으로 직접 인라인 처리:
    del_lam = V_emit / c_kms * lambda_c
    Gaussian_Fitting_Flux = (Llambda_max - Lc) * np.exp(
        -(lambda_cm - lambda_c)**2 / del_lam**2) + Lc
  기능적으로 동일하고 더 빠르며 가독성도 향상.

------------------------------------------------------------------------
C-06  Spectrum_RT — 배열 마스킹
------------------------------------------------------------------------
기존 방식:
  ioc = np.where(lam <= lam_c)[0]   # 중간 인덱스 배열 생성

새로운 방식:
  mask = lam <= lam_c                # 불리언 마스크 직접 사용
  return lam[mask], spec_tot[mask], spec_halo[mask]

불리언 인덱싱이 np.where 대비 약간 빠르고 메모리 할당도 줄어듦.

------------------------------------------------------------------------
C-07  _read_ele_column 중복 정의 제거
------------------------------------------------------------------------
기존 방식:
  _read_ele_column 이 파일 내에 두 번 정의됨:
    (a) CLOUDY_data_path 안의 클로저 (인자 3개)
    (b) 파일 하단의 모듈 함수 (인자 4개, read_file에서 사용)
  어느 쪽을 수정해도 다른 쪽에 반영되지 않는 유지보수 위험.

새로운 방식:
  단일 모듈 수준 함수 하나로 통합. 두 호출 지점 모두 같은 함수 사용.

------------------------------------------------------------------------
C-08  CLOUDY_data_path — 미사용 변수 제거
------------------------------------------------------------------------
기존 방식:
  N_HI, N_HII, N_CIV, frac_O, frac_N, frac_Mg 를 계산했으나
  반환값에도 없고 내부에서도 사용되지 않았음.

새로운 방식:
  미사용 변수 전부 제거. 반환 dict 는 완전히 동일.
  불필요한 메모리 할당과 코드 혼란 감소.

------------------------------------------------------------------------
C-09  numba 없을 때 graceful fallback
------------------------------------------------------------------------
새로 추가:
  import 블록에서 numba 임포트를 try/except 로 감쌈.
  numba를 가져오지 못하면:
    - 아무것도 하지 않는 더미 @njit 데코레이터 자동 정의.
    - prange 를 내장 range 로 대체.
    - _abel_kernel → linspace + np.interp 를 사용한 numpy 벡터화 버전으로 대체.
    - _photon_number_kernel → 완전 벡터화 numpy 버전으로 대체.
  어느 서버에서도 코드가 정상 동작하며, numba 유무에 따라 속도만 차이남.

4. 예상 성능
================================================================================

  전형적인 작업 흐름에서 가장 큰 비용은 CLOUDY_data_path() 호출이며,
  내부에서 Surface_Brightness_CLOUDY 를 CIV, HeII, Lya 3회 호출함.

  CLOUDY_data_path 1회 호출 기준 예상 실행 시간:

    항목                         RT.py (기존)   RT_v1.py (신)    속도 향상
    ────────────────────────────────────────────────────────────────────
    Surface_Brightness_CLOUDY     약 3~10초       약 0.05~0.3초   20~100배
      (3종 × 70 R × quad 적분)      (병렬 numba Abel)
    _read_ele_column (×8회)        약 0.1초        약 0.01초       약 10배
    SED_properties                 약 0.01초       약 0.01초        ~1배
    photon_number_SB               약 0.001초      <0.001초         약 5배
    ────────────────────────────────────────────────────────────────────
    전체 (대략)                    약 3~10초        약 0.1~0.5초   10~50배

  참고사항:
  - 설치 후 첫 번째 호출 시 numba JIT 컴파일 (~5~10초) 이 한 번 발생하고
    이후는 캐시(__pycache__)에서 바로 로드됨.
  - 실제 속도 향상은 CPU 코어 수에 따라 다름 (prange 병렬화).
  - n_pts=512 기본값은 일반적인 CLOUDY emissivity 프로파일에서
    scipy quad 대비 오차 0.1% 미만. 프로파일이 매우 가파를 경우 1024 사용.
